--- dmypy ---
mypy(파이썬 코드의 타입오류를 검사해주는 도구)를 빠르게 돌리기 위한 백그라운드 서버 모드, mypy는 실행할 때마다 전체 프로젝트를
다시 분석하기 때문에 느리고 큰 프로젝트에서 오래 걸림, 그래서 dmypy(daemon: 백그라운드 프로세서 + mypy)가 나오게 되었고,
코드 상태를 계속 기억하며, 변경된 부분만 검사하기 때문에 빠르다. 그래서 큰 프로젝트나 타입검사가 자주 필요하거나 개발 속도가
중요한 곳에 사용 된다.

--- classmethod, 바다코끼리 ---
classmethod(객체가 아니라 클래스 자체를 다루는 메서드): 팩토리 메서드를 만들 때, 객체 생성 로직 커스터마이징 할 때 사용
바다코끼리(:=): 한 줄로 코드를 줄임(코드 간결), while 루프, 조건문 안에서 값 저 필요할 때 사용

--- TDD(Test Driven Development) ---
테스트를 먼저하고 실제 코드를 만드는 방식이다.
Red: 실패하는 테스트 작성(기능이 없으므로 테스트는 당연히 실패함)
Green: 테스트를 통과하도록 최소 구현
Refactor: 코드 정리(중복 개선, 구조 개선, 가독성 향상)
사용이유: 버그 감소, 설계가 명확, 리팩토링 안전, 요구 사항이 텍스트로 문서화 됨

--- Service Design Pattern ---
비지니스 로직을 Service 계층에 모으는 설계 방식이다. 이 방식은 Controller를 가볍게 하고 (역할 분리, 테스트 쉬움, 재사용 가능)
실제 일은 Service가 하도록 한다.

--- Repository Design Pattern ---
DB 접근 코드를 모아두는 설계 방식(Service 안에서 직접 DB 쿼리를 다루면 DB코드가 여기저기 흩어지게 되고, DB 변경시 모두 수정해야
하며 테스트가 어려워짐) 정확하게는 역할 분리를 위해서다 (DB 교체 쉬움, 가독성이 좋고, 테스트에서 Mock 가능, 계층분리 등이 있다.)

--- DTO(Data Transfer Object) ---
데이터를 전달하기 위해 사용하는 객체(필요한 필드만 전달, 내부 구조 숨김: 비밀번호 노출 등 보완 문제, 보안 강화)이며 내부 모델을 보호,
API 응답 구조 통제, 프론트와 계약 명확화, 유지보수 용이한 목적으로 사용되고 있다.
